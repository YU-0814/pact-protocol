Implementation

How ClinicalTrials.gov Could Adopt PACT

Step 1: Add /.well-known/pact.json (5 minutes, L1)

{
  "pact": "1.0",
  "site": "clinicaltrials.gov",
  "schemas": ["pact:health/clinical-trial@1"],
  "endpoints": {
    "pact:health/clinical-trial@1": {
      "list": "/pact/studies",
      "item": "/pact/studies/{nctId}",
      "search": "/pact/studies?q={query}"
    }
  },
  "rate_limit": { "rpm": 120 },
  "license": { "ai_input": true, "ai_train": true, "attribution": true }
}

Step 2: Serve PACT responses from existing API (hours, L3)

ClinicalTrials.gov already has a JSON API (/api/v2/studies). A thin PACT wrapper would compress the verbose 45,000-token API response into 850 tokens using key compression and the pact:health/clinical-trial@1 schema.

Result: AI medical assistants worldwide could instantly discover and consume clinical trial data in a standardized, token-efficient format — no scraping, no API documentation, no custom parsing per site.
