더 많은 활용 시나리오

PACT가 가장 큰 차이를 만드는 곳

AI 에이전트에게 구조화 데이터가 필요한 모든 사이트 — HTML 장식이 아닌.

clinicaltrials.gov → pact:health/clinical-trial@1

임상시험 레지스트리

AI 의료 어시스턴트가 환자에게 맞는 임상시험을 찾습니다. HTML 페이지는 CSS 프레임워크와 용어 사이드바 아래에 데이터를 묻어둡니다.

59x 절감5% 유용
drugbank.com → pact:health/drug-interaction@1

약물 상호작용 DB

AI 약사가 실시간 약물 상호작용 확인을 위해 구조화 데이터가 필요합니다. DrugBank 페이지는 복잡한 네비게이션과 유료벽이 있습니다.

~40x 추정8% 유용
coupang.com → pact:commerce/product@1

한국 이커머스

AI 쇼핑 에이전트가 쿠팡, 네이버, 11번가 가격을 비교합니다. 상품 페이지는 추천, 리뷰 UI, 추적 코드로 비대합니다.

57x 실측2% 유용
zillow.com → pact:realestate/listing@1

부동산 목록

AI 에이전트가 집을 찾아줍니다. Zillow 페이지는 지도, 3D 투어, 대출 계산기를 로드하지만 에이전트는 가격, 방, 면적만 필요합니다.

~45x 추정3% 유용
pubmed.ncbi.nlm.nih.gov → pact:health/research-paper@1

PubMed 연구 논문

AI 연구 어시스턴트가 특정 결과를 찾습니다. PubMed HTML에는 관련 논문, 인용 UI 등 전체 페이지 크롬이 포함됩니다.

~35x 추정10% 유용
data.seoul.go.kr → pact:local/public-data@1

서울 열린데이터광장

AI 시민 에이전트가 도시 통계를 필요로 합니다. 포털은 단순한 데이터를 무거운 정부 CMS 템플릿으로 감쌉니다.

~50x 추정4% 유용

구현 방법

ClinicalTrials.gov가 PACT를 도입한다면

1단계: /.well-known/pact.json 추가 (5분, L1)

{
  "pact": "1.0",
  "site": "clinicaltrials.gov",
  "schemas": ["pact:health/clinical-trial@1"],
  "endpoints": {
    "pact:health/clinical-trial@1": {
      "list": "/pact/studies",
      "item": "/pact/studies/{nctId}",
      "search": "/pact/studies?q={query}"
    }
  },
  "rate_limit": { "rpm": 120 },
  "license": { "ai_input": true, "ai_train": true, "attribution": true }
}

2단계: 기존 API에서 PACT 응답 제공 (수 시간, L3)

ClinicalTrials.gov는 이미 JSON API (/api/v2/studies)가 있습니다. 얇은 PACT 래퍼를 추가하면 장황한 45,000 토큰 API 응답을 키 압축과 pact:health/clinical-trial@1 스키마를 사용해 850 토큰으로 압축할 수 있습니다.

결과: 전 세계 AI 의료 어시스턴트가 즉시 임상시험 데이터를 표준화된, 토큰 효율적인 형식으로 발견하고 소비할 수 있습니다 — 스크래핑 없이, API 문서 없이, 사이트별 맞춤 파싱 없이.

데이터 출처 및 저작자 표시:

임상시험 구조는 ClinicalTrials.gov (미국 국립의학도서관)에서 영감을 받았습니다. 모든 데모 시험 데이터는 합성 데이터이며 실제 환자 데이터는 사용되지 않았습니다.

토큰 수치는 2026년 3월에 수행된 실제 페이지 분석을 기반으로 합니다.

기타 참조 사이트(DrugBank, 쿠팡, Zillow, PubMed, 서울 열린데이터)는 비교 설명 목적으로만 사용되었습니다.
