PACT 프로토콜 스펙 문서

버전 1.0 — PACT 워킹 그룹 2026

English | 한국어

이것은 PACT 스펙의 요약입니다. 전체 규격은 GitHub 저장소를 참조하세요.

개요

PACT (Protocol for Agent Content Transfer)는 AI 에이전트에게 구조화된, 토큰 효율적인 콘텐츠를 제공하는 개방형 표준입니다. 탐색 표준(robots.txt, llms.txt)과 상호작용 프로토콜(MCP, A2A) 사이의 빈자리를 채웁니다.

MCP가 AI 에이전트에게 도구를 제공하고 A2A가 에이전트를 연결한다면, PACT는 콘텐츠 전달 문제를 해결합니다: 웹사이트가 자신의 실제 데이터(상품, 기사, 목록)를 컴팩트하고, 타입이 지정된, 액션 가능한 형식으로 AI 에이전트에게 제공하는 방법.

설계 원칙

  • 토큰 효율성 — AI 모델이 소비하는 토큰 최소화 (HTML 대비 57배 절감)
  • 점진적 채택 — JSON 파일 하나로 시작, 점진적으로 확장
  • 스키마 기반 — 버전이 지정된 스키마로 안정적인 파싱
  • 게시자 통제 — 사이트 운영자가 조건, 가격, 접근 규칙을 정의
  • 크로스 플랫폼 — 웹, 모바일, IoT 등 HTTP가 통하는 어디서나

아키텍처

PACT는 5단계 스택을 사용하며, 각 레이어는 이전 레이어 위에 구축됩니다. 점진적으로 채택할 수 있습니다.

레이어이름목적핵심 파일 / 필드
0Discovery에이전트가 PACT 엔드포인트를 찾습니다/.well-known/pact.json
1Schema데이터 타입이 식별되고 버전이 지정됩니다pact:domain/type@version
2Data구조화된 JSON 콘텐츠 응답$pact, $s, items
3Media이미지 및 비디오 참조데이터 필드 내 URL 포인터
4Actions트랜잭션 작업$actions 배열

레이어 0: 디스커버리

/.well-known/pact.json에 위치한 JSON 파일로, 에이전트에게 사이트가 어떤 콘텐츠를 제공하고, 어떤 스키마를 사용하며, 어디서 데이터를 요청할 수 있는지 알려줍니다.

레이어 1: 스키마

모든 PACT 응답은 pact:domain/type@version 형식의 스키마 URI로 데이터 타입을 식별합니다. 에이전트는 이를 통해 추측 없이 구조를 파싱하고 이해합니다.

레이어 2: 데이터

실제 콘텐츠 — 타입이 지정된 필드가 있는 구조화된 JSON. 최대 토큰 효율성을 위해 객체 모드(items 배열)와 테이블 모드(cols + rows) 모두 지원합니다.

레이어 3: 미디어

미디어 에셋(이미지, 비디오, 문서)은 임베드 대신 URL로 참조되어 응답 페이로드를 작게 유지하고 에이전트가 선택적으로 가져올 수 있게 합니다.

레이어 4: 액션

$actions 배열로 사이트가 구매, 예약, 구독 등의 트랜잭션 작업을 정의된 파라미터와 함께 타입이 지정된 엔드포인트로 노출할 수 있습니다.

디스커버리 프로토콜

모든 PACT 지원 사이트는 /.well-known/pact.json에 디스커버리 문서를 호스팅합니다. 이것이 AI 에이전트의 진입점입니다.

필수 필드

필드타입설명
pactstring프로토콜 버전 (예: "1.0")
sitestring도메인 이름 (예: "example.com")
schemasstring[]지원하는 스키마 URI 목록
endpointsRecord스키마 ID를 엔드포인트 정의(list, item, search)에 매핑

선택 필드

필드타입설명
descriptionstring사이트 설명
platformsRecord플랫폼별 설정 (web, ios, android)
rate_limitobject속도 제한: rpm, burst
authobject인증: type, register
licenseobject사용 조건: ai_input, ai_train, attribution
conformancestring자체 선언 등급: L1, L2, L3, L4

예제

{
  "pact": "1.0",
  "site": "store.example.com",
  "description": "온라인 전자제품 스토어",
  "schemas": ["pact:commerce/product@1"],
  "endpoints": {
    "pact:commerce/product@1": {
      "list": "/pact/products",
      "item": "/pact/products/{id}",
      "search": "/pact/products?q={query}"
    }
  },
  "rate_limit": { "rpm": 60, "burst": 10 },
  "license": { "ai_input": true, "ai_train": false, "attribution": true }
}

스키마 시스템

PACT 스키마는 pact:domain/type@version URI 형식을 따르며, 데이터 타입에 대한 범용 명명 규칙을 제공합니다.

스키마 URI 형식

pact:domain/type@version

예제:
  pact:commerce/product@1
  pact:news/article@1
  pact:food/restaurant@1
  pact:media/video@1

표준 도메인

도메인범위예제 타입
commerce이커머스, 상품product, price-comparison, store
news기사, 뉴스article, headline, feed
food음식점, 메뉴restaurant, menu, recipe
local지역 비즈니스business, service, review
media미디어 콘텐츠video, podcast, channel
travel여행, 숙박hotel, flight, booking
events이벤트, 행사event, ticket, venue
education교육, 강좌course, program, cert
jobs채용, 커리어posting, company, salary

데이터 레이어

PACT 응답은 토큰 효율성을 위해 설계된 최소 엔벨로프 형식을 사용합니다. 두 가지 모드를 지원합니다:

객체 모드

items 배열이 있는 표준 JSON. 토큰 효율을 위한 압축 키.

{
  "$pact": "1.0",
  "$s": "pact:commerce/product@1",
  "$t": 1710590400,
  "$ttl": 300,
  "items": [
    { "id": "p1", "n": "위젯", "p": 9.99, "stk": true },
    { "id": "p2", "n": "가젯", "p": 19.99, "stk": true }
  ],
  "total": 2,
  "page": { "offset": 0, "limit": 20 }
}

테이블 모드

열 + 행으로 최대 압축. 대량의 균일한 데이터셋에 최적.

{
  "$pact": "1.0",
  "$s": "pact:commerce/product@1",
  "$t": 1710590400,
  "$layout": "table",
  "cols": ["id", "n", "p", "stk"],
  "rows": [
    ["p1", "위젯", 9.99, true],
    ["p2", "가젯", 19.99, true]
  ],
  "total": 2
}

시스템 필드

필드필수설명
$pact예프로토콜 버전
$s예스키마 URI (pact:domain/type@version)
$t예응답 생성 Unix 타임스탬프
$ttl아니오캐시 TTL (초)
$layout아니오열 기반 형식은 "table"로 설정
total아니오매칭 항목의 총 개수
page아니오페이지네이션: offset, limit, next

액션

$actions 배열로 사이트가 PACT 응답 내에 직접 트랜잭션 엔드포인트를 노출할 수 있습니다. AI 에이전트가 사용자를 대신하여 구매, 예약, 구독 등의 작업을 수행할 수 있게 합니다.

액션 구조

액션은 스펙에 따라 verb (HTTP 메서드)와 name (액션 식별자)를 사용합니다:

{
  "actions": [
    {
      "verb": "POST",
      "name": "purchase",
      "description": "장바구니에 추가",
      "url": "/pact/cart/add",
      "body": { "product_id": "string", "quantity": "number" },
      "auth_required": true
    },
    {
      "verb": "POST",
      "name": "reserve",
      "description": "예약하기",
      "url": "/pact/reservations",
      "body": { "date": "string", "time": "string", "party_size": "number" }
    }
  ]
}

표준 액션 타입

  • purchase — 상품 또는 서비스 구매
  • reserve — 예약하기
  • subscribe — 구독 시작
  • contact — 메시지 또는 문의 보내기
  • bookmark — 항목 저장 또는 즐겨찾기

적합도 등급

PACT는 점진적 채택을 위해 4개의 적합도 등급을 정의합니다. 각 등급은 이전 등급을 포함합니다.

등급이름요구사항소요 시간
L1탐색 가능/.well-known/pact.json에 유효한 pact.json~5분
L2타입 지정L1 + 스키마 식별자, $pact와 $s가 포함된 구조화 응답~30분
L3효율적L2 + 테이블 모드, 페이지네이션, 압축수 시간
L4신뢰L3 + 인증, $actions, 라이선스, 에이전트 검증수 일

크로스 플랫폼 지원

PACT 응답은 $platforms 필드를 사용하여 대상 플랫폼을 선언할 수 있습니다. 에이전트가 가능한 경우 플랫폼별 콘텐츠를 전달할 수 있습니다.

{
  "$pact": "1.0",
  "$s": "pact:local/restaurant@1",
  "$platforms": ["web", "ios", "android"],
  "items": [...]
}

지원 플랫폼

  • web — 브라우저 기반 웹 애플리케이션
  • ios — Apple iOS 애플리케이션
  • android — Android 애플리케이션
  • api — 헤드리스 API 컨슈머
  • iot — IoT 디바이스 및 임베디드 시스템

보안 고려사항

PACT는 L4 적합도에서 안전한 콘텐츠 전달을 위한 조항을 포함합니다:

  • 인증 — 보호된 엔드포인트를 위한 API 키, OAuth 2.0, 커스텀 인증
  • 에이전트 검증 — 요청 에이전트의 신원과 권한 확인
  • 속도 제한 — 에이전트 또는 키별 요청 빈도 제어
  • 콘텐츠 라이선스 — 디스커버리에서 사용 조건과 저작자 표시 요구사항 정의
  • HTTPS 필수 — 모든 PACT 엔드포인트는 TLS 암호화를 사용해야 합니다
